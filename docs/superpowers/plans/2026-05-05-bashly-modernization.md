# Bashly Modernization Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Refactor `src/bashly.yml` (currently 750 lines, ~500 of duplication) into a DRY, multi-file Bashly project with declared metadata, generated zsh completions, and a no-network bats test harness.

**Architecture:** Split `bashly.yml` into per-group files (`commands/install.yml`, `commands/uninstall.yml`, `commands/setup.yml`, `commands/outdated.yml`) using Bashly's `import:` directive. Use YAML anchors to deduplicate identical `--version` / `--force` flag blocks across ~44 simple install commands. Add root-level `environment_variables`, `footer`, and `dependencies`. Generate completions with `bashly add comp_function`. Add bats test harness covering registry resolver, version helpers, and build smoke tests — no network calls.

**Tech Stack:** Bashly 1.1.9, Bash 5+, Bats-core 1.x, YAML, GNU diff.

**Spec:** `docs/superpowers/specs/2026-05-05-bashly-modernization-design.md`

## Task 2: Introduce YAML anchors for `std_install_flags`

**Files:**
- Modify: `src/commands/install.yml` (add anchor, replace ~44 simple flag blocks with alias)

**Verify by:** byte-identical `gt` after regeneration. YAML anchors do not change emitted parser behavior — only source-text size.

- [ ] **Step 1: Save current baseline**

```bash

```

- [ ] **Step 2: Add anchor to top of `src/commands/install.yml`**

Insert at the very top of the file (above `name:`):

```yaml
x-anchors:
  std_install_flags: &std_install_flags
  - long: --version
    short: -v
    arg: version
    help: "Version to install (e.g., latest)"
    default: latest
  - long: --force
    short: -f
    help: "Force overwrite if already installed"
```

Note: `x-anchors` is a YAML key Bashly ignores (it's not a recognized command field). Anchors defined here are reusable elsewhere in the same file.

**Important:** Verify the anchor flag values *exactly* match what each tool currently has. The current per-tool `help:` strings vary slightly (`"kubectl version to install (e.g., v1.30.0 or latest)"` vs `"k9s version to install (e.g., v0.32.4 or latest)"`). The anchor will lose these per-tool example versions in `help:`. Accept this loss — generic help text is acceptable; example versions belong in `examples:` (Task 5).

- [ ] **Step 3: Replace simple-tool flag blocks with alias**

For each of the ~44 tools in `commands/install.yml` whose flags are exactly `--version` (default `latest`) + `--force` and nothing else, replace the inline flag list with `flags: *std_install_flags`. Example before:

```yaml
- name: k9s
  help: Install k9s
  flags:
  - long: --version
    short: -v
    arg: version
    help: k9s version to install (e.g., v0.32.4 or latest)
    default: latest
  - long: --force
    short: -f
    help: Force overwrite if k9s is already installed
```

After:

```yaml
- name: k9s
  help: Install k9s
  flags: *std_install_flags
```

**Do NOT convert these special-case tools** (their flags differ — keep verbatim):
- `bashly` (has `dependencies: [gem]`)
- `claude` (has `dependencies: [npm]`)
- `gem` (only `--force`, no `--version`)
- `skopeo` (only `--force`)
- `tmux` (only `--force`)
- `rsync` (only `--force`)
- `install multi` (different flag set entirely)

Tools requiring conversion (44 total): kustomize, k9s, kubectl, fastfetch, direnv, btop, zsh, antidote, thefuck, argocd, velero, kubeconform, krew, kubent, sops, age, trivy, kubens, kubecolor, popeye, kube-linter, kyverno, conftest, vault, kubeseal, zoxide, eza, bat, ripgrep, starship, fzf, fd, mcfly, infisical, ksops, helmfile, helm, chezmoi, trino, nerdctl, viddy, hyperfine, jq, yq, gum, uv.
